# KIS Trading Alert Bot

한국투자증권 OpenAPI 기반 투자 알림 봇.  
보유 종목 변동, 자산 리포트, ETF 브리핑, 수급 분석을 **Discord + Telegram**으로 자동 전송합니다.

## 어떤 알림을 받을 수 있나요?

### 📌 매수/매도 체결 알림 (실시간)
주식을 사거나 팔면 **60초 이내에 알림**이 옵니다.
- 국내 주식: 장중(09~15시)에만 감지
- 해외 주식: 24시간 감지 (미국/일본/홍콩/중국)
- 수량 변화가 없으면 알림을 보내지 않아 스팸 방지

### 📊 08:30 장 시작 전 종합 브리핑
**매일 아침 08:30**에 자산 현황 + 수급 + 🎯 **신규 종목 발굴 리포트**를 받습니다.

포함 정보:
- **보유 종목별 수익률** — 평균 매입가 vs 현재가, 손익금, 수익률
- **총 자산** — 보유종목 평가 + 예수금 (KIS API output2 기반)
- **실현손익** — 매도 확정 순수익 (올해 / 최근 10년, KIS API 기반)
- **기간 수익금 + 예상 수익률** (아래 "수익금/수익률 계산 방식" 참조)
- **수급 TOP 3** — 당일 외국인·기관 순매수 상위 종목
- **ETF 거래량 TOP 3** — 당일 가장 많이 거래된 ETF
- **🎯 신규 종목 발굴 브리핑** — 복합 지표 기반 고품질 매수 후보 (최대 5종목)

## 🎯 신규 종목 발굴 브리핑 (08:30)

**보유 여부와 무관하게** 시장 전체를 스캔해서 매수 검토 가치가 있는 종목을 추려줍니다.
품질 기준을 충족하지 못하는 날에는 "오늘은 조건 충족 종목 없음" 한 줄만 표시 — 억지 추천을 하지 않습니다.

### 후보 풀 소스 & 사전계산

**Naver Finance 거래량 상위** (sise_quant) KOSPI+KOSDAQ 각 2페이지(~200종목)를 스크래핑해 후보 풀을 확보합니다. ETF/ETN/스팩/우선주/초저가주(<1,000원)/거래대금 100억 미만은 자동 제외. 거래대금 상위 40종목을 심화 분석 대상으로 선정합니다.

> **왜 Naver인가?** KIS `foreign-institution-total` API(FHPTJ04400000)가 계정에 따라 0건을 반환하는 경우가 있어, 외부 의존 없이 안정적으로 후보를 확보하기 위해 Naver Finance를 primary 소스로 전환했습니다. KIS 수급 데이터(Redis `FRGN_FLOW`/`ORGN_FLOW`)는 축적되면 **보너스 가산점**으로 활용됩니다.

**매 거래일 16:00** 장마감 종합 분석 잡이 발굴 후보를 사전계산해 `DISCOVERY_SNAPSHOT:{YYYYMMDD}` (10일 보관)에 저장. 다음날 08:30 브리핑은 **최근 3거래일 이내 스냅샷을 우선 로드**하고, 없으면 라이브 호출로 폴백합니다.

**스캔 범위**: KOSPI + KOSDAQ 거래량 상위 200종목 → 필터링 후 상위 40종목 심화 분석

## 🧪 추천 종목 검증 & 자기보정

추천한 종목이 **실제로 올랐는지**를 추적하고, 쌓인 데이터로 점수 계산을 자기보정합니다.

### 1. 검증 추적
- 16:00 장마감 잡이 실행될 때, 과거 추천(`DISCOVERY_RECO:{date}:{code}`)을 스캔해 **+1d / +3d / +5d / +10d** 가격을 각 지평선마다 한 번씩 기록합니다.
- 각 레코드에는 추천 당시 점수, 지표 근거 목록, 진입가, 확인일별 수익률이 누적됩니다 (45일 보관).
- 16:05에도 안전망으로 한 번 더 `job_verify_recommendations()`이 실행됩니다 (장마감 잡 실패 대비 중복 안전).

### 2. 지표별 성과 통계 (`DISCOVERY_INDICATOR_STATS`)
- 숫자/연속일수 같은 가변 부분을 정규화(`N`으로 치환)해 같은 지표를 하나의 버킷으로 모읍니다.
  - 예: `수급:외국인 5일 연속 순매수` 와 `수급:외국인 3일 연속 순매수` → 같은 버킷
- 버킷마다 `{count, wins, sum_ret}`이 누적되어 **승률과 평균 +5d 수익률**을 항상 계산할 수 있습니다.

### 3. 자동 가중치 보정
- `get_indicator_reliability(label)` → 샘플 ≥ 10건 기준:
  - 승률 < 40% 또는 평균수익 < −1% → **0.5배** (점수에서 -1.5점)
  - 승률 > 65% & 평균수익 > +1% → **1.2배** (점수에서 +0.6점)
  - 그 외는 1.0배 (중립)
- 이 가중치는 발굴 후보의 최종 점수에 **자동 반영**되어, 시간이 지나면 잘 맞는 지표 조합이 더 높은 점수를 받고 꾸준히 빗나가는 지표는 자연 도태됩니다.

### 4. 주간 검증 리포트 (매주 금요일 16:00 장마감 분석에 포함)
```
🧪 [발굴 추천 검증 리포트]
  지평선 | 표본 | 승률 | 평균수익
  +1d    |   47 |  55% | +0.82%
  +3d    |   45 |  62% | +1.45%
  +5d    |   42 |  68% | +2.11%
  +10d   |   35 |  60% | +1.98%

  🏆 지표 상위 (샘플≥10, 평균수익 기준)
    · 수급:외국인+기관 동시 순매수 — 38건, 71% 승, +2.65%
    · 추세:정배열 (종가>MA20>MA60) — 27건, 67% 승, +2.10%
    · 거래량:N배 (N일평균 대비) — 22건, 63% 승, +1.72%
  ⚠️ 지표 하위
    · 기술:RSI N (과매수) — 12건, 33% 승, −1.80%
```

## ✍️ 수동 입출금 명령 (텔레그램 / 디스코드)

외부 자금이동이 발생했을 때 **텔레그램이나 디스코드 채널에 한 줄 명령만** 보내면 잔고 현황 Redis에 바로 반영됩니다.

### 명령어
```
/입금 500000              → +50만원 입금 기록
/입금 1,500,000 배당       → 라벨 포함
/출금 200000 생활비        → -20만원 출금 기록
/deposit 1000000
/withdraw 500000
```

### 반영 범위
1. `MANUAL_CASHFLOW:{YYYYMMDD}` 리스트에 JSON 엔트리 추가 (영구 기록).
2. `MANUAL_CASHFLOW_LOG` 마스터 리스트에 마지막 200건 유지.
3. **`INITIAL_ASSETS_{year}` 기준자산 자동 보정** — 입금액만큼 기준을 올리고 출금액만큼 내려서 `예상 수익률` 분모가 왜곡되지 않도록 합니다.
4. 08:30 브리핑에 `✍️ [수동 입출금 기록]` 섹션으로 최근 10건 표시.

### 작동 방식
- **텔레그램**: 기존 `TELEGRAM_BOT_TOKEN`으로 `getUpdates` long-poll 스레드를 추가로 띄웁니다. `TELEGRAM_CHAT_ID`와 일치하는 채팅방에서 온 명령만 처리 (다른 곳에서 오는 명령은 무시).
- **디스코드**: Webhook은 수신 불가이므로, 명령을 받으려면 **별도 Bot Token + Channel ID** 가 필요합니다. `DISCORD_BOT_TOKEN`과 `DISCORD_CHANNEL_ID` 환경변수를 설정하면 해당 채널의 메시지를 15초 주기로 폴링합니다. (미설정이면 디스코드 수신 기능은 자동 비활성화)

### 보안
- `chat_id` / `channel_id` 매칭되지 않는 메시지는 드롭됩니다.
- 봇 자신의 메시지(author.bot)는 무시합니다.
- 명령 파서는 `/(입금|출금|deposit|withdraw) <금액> [라벨]` 정규식만 허용합니다.

## 💰 수익금 / 수익률 계산 방식

기간 중 입출금이 발생해도 수익률이 왜곡되지 않도록 **입출금 독립** 공식을 사용합니다.

### 수익금 (정확)
```
수익금 = 기준일 이후 실현손익(KIS TTTC8715R) + (현재 미실현 − 기준일 미실현)
```
realized와 ΔU 둘 다 cash flow에 영향받지 않으므로 입금/출금 유무와 무관하게 정확합니다.

### 예상 수익률 (간이식, 현재)
```
예상 수익률 = 수익금 / 기준일 총자산 × 100
```
- "기준일"은 봇이 해당 연도 처음 관측한 시점 (Redis `INITIAL_ASSETS_{year}`, `INITIAL_UNREALIZED_{year}`, `INITIAL_DATE_{year}`)
- **예상**이라고 표기하는 이유: 기간 중 입출금에 따른 분모 보정이 없습니다

### 외부 자금이동 자동 감지 (오차 0)
매일 08:30·16:00 브리핑 때마다 `ASSET_SNAPSHOT:{YYYYMMDD}` 키로 일별 총자산/미실현/누적실현 스냅샷을 저장합니다. 스냅샷이 2일치 이상 쌓이는 시점부터 **회계 항등식 기반 역산**으로 입출금(+배당 수령)을 자동 감지합니다.

```
CashFlow[D] = Δtotal − Δrealized_cum − ΔU
```

**왜 오차 0인가**
- `Δtotal`: KIS 단일 snapshot의 총자산 차이 — 빠지는 돈 없음
- `Δrealized_cum`: `TTTC8715R`가 반환하는 **수수료·세금 차감 후 순 실현손익**
- `ΔU`: 평가손익 합계 차이 — 시장 변동만 포함, 외부 현금흐름 무관
- 세 항목 차감 후 남은 잔여분 = 외부 유입/유출 (순수 회계 항등식)

**라벨링**: 배당금/권리 수령도 이 버킷에 포함돼 **"순 자금이동 (입출금+배당)"**으로 표시. 숫자 자체의 정확도는 100%이고, "입출금"과 "배당"의 분리만 하지 않습니다.

**08:30 브리핑 섹션 예시**
```
💸 [외부 자금이동] 기준일 이후 자동 감지 (입출금+배당 합산)
  🟢 순 자금이동: +500,000원
    ↳ 유입 합계: +700,000원
    ↳ 유출 합계: -200,000원
  · 관측 일수: 14일 (20260414~20260427)
  · 최근 이동 내역:
    🟢 20260421: +500,000원
    🔴 20260425: -200,000원
  ※ 회계 항등식 기반 역산 (오차 0). 배당/권리 수령도 포함
```

| 08:10 | 월간 ETF 리포트 (매월 첫 거래일) |
| 08:20 | 외국인/기관 수급 추세 TOP |
| 08:30 | 신규 ETF 2차 체크 (장전시간외) |
| 08:30 | 종합 브리핑 (자산/수급/ETF) + ✍️ 수동 입출금 + 🎯 신규 종목 발굴(사전계산 스냅샷) |
| 15:50 | 외국인/기관 수급 스냅샷 저장 |
| 16:00 | 장마감 종합 분석 → 🎯 **발굴 후보 사전계산** + 🧪 **과거 추천 검증** (금요일엔 검증 리포트 포함) |
| 16:05 | 검증 잡 안전망 (과거 추천 +1d/+3d/+5d/+10d 수익률 갱신) |
| 매 60초 | 잔고 변동 감지 (매수/매도 알림) |
| 상시 | 텔레그램 long-poll + (옵션) 디스코드 폴링 → `/입금` `/출금` 명령 수신 |

## 설치 및 실행

### 필요한 것
- Python 3.10+
- KIS OpenAPI 앱키/시크릿 ([한국투자증권 OpenAPI](https://apiportal.koreainvestment.com/))
- Discord Webhook URL 또는 Telegram Bot Token + Chat ID
- (선택) DART API 키 ([opendart.fss.or.kr](https://opendart.fss.or.kr/))
- (선택) Redis — Upstash 무료 티어 추천

### 설치
```bash
git clone https://github.com/lirongzzuin/KIS_Discord_Alert_Bot.git
cd KIS_Discord_Alert_Bot
pip install -r requirements.txt
cp .env.example .env  # 환경변수 편집
python kis_discord_alert.py
```

## fly.io 배포 (무료)

```bash
# 1. 앱 생성
flyctl apps create kis-discord-alert-bot --org personal

# 2. Redis 생성
flyctl redis create --name kis-alert-redis --region nrt --no-replicas --enable-eviction -o personal

# 3. 시크릿 등록
flyctl secrets set \
  KIS_APP_KEY="..." KIS_APP_SECRET="..." KIS_ACCOUNT_NO="..." \
  DISCORD_WEBHOOK_URL="..." TELEGRAM_BOT_TOKEN="..." TELEGRAM_CHAT_ID="..." \
  DART_API_KEY="..." REDIS_URL="..." \
  --app kis-discord-alert-bot

# 4. 배포
flyctl deploy

# 5. 로그 확인
flyctl logs --app kis-discord-alert-bot --no-tail | tail -20
```

## 기술 스택

- **Python 3.11** + schedule + requests + redis + holidays
- **KIS OpenAPI** — 잔고, 실현손익, 수급, 환율 조회
- **DART OpenAPI** — ETF 상장 예정 (증권신고서 조회 + 문서 파싱으로 상장예정일 추출)

- **Discord Webhook + Telegram Bot API** — 알림 전송 (마크다운 링크 지원)
- **Upstash Redis** — 스냅샷, 캐시, 수급 추세 데이터
- **fly.io** — 무료 클라우드 배포 (shared-cpu-1x, 256MB)
